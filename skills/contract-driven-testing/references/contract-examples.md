# Contract Examples (for this repository)

These are examples of `*.contract.md` already present in this repo:

- `docs/requirements/TimeSys.contract.md`
- `docs/requirements/RefSys.contract.md`

When mapping clauses to tests, look for:
- Round-trip / inversion requirements
- Representation consistency requirements
- Invariants (orthogonality, determinant, monotonicity, etc.)
- Failure semantics (missing data, invalid ranges, out-of-domain)
- External reference cross-check requirements (e.g., SOFA)
