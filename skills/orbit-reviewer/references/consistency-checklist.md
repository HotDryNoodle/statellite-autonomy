## Consistency Checklist (Theory ↔ Code ↔ Tests)

目标：用最少的步骤发现“三者不一致”的断点。每条检查项都应能指向具体证据（段落/公式、文件/函数/变量、测试用例）。

### A. Theory ↔ Definitions

- 是否有**符号表**：每个符号的物理意义、单位、坐标系、角度与时间尺度约定。
- 是否存在**同名异义**：例如 \u03a9 既作 RAAN 又作地球自转角；n 既作平均角速度又作法向量。
- 是否明确**输入输出**：状态量是 \u007b\u03c1,\u03bd\u007d 还是轨道根数？观测是量测值还是残差？

### B. Definitions ↔ Code

- 单位是否显式：m vs km；s vs day；rad vs deg。
- 坐标系是否显式：ECI/ECEF/RTN/LVLH；是否发生隐式变换（例如使用地固速度却按惯性方程积分）。
- 角度范围与分支：atan2 结果、\u03bd/E/M 的分支、i \u2248 0 或 e \u2248 0 的退化处理是否与论文定义一致。
- 常量来源：\u03bc、J2、R_e、\u03c9_E、AU 等常量是否与文档一致（数值、单位、参考历元）。

### C. Formula ↔ Implementation

对每个关键公式做“逐项对齐”：

- 项是否缺失/多余：例如 J2 摄动项的系数与符号。
- 向量/矩阵维度是否一致：转置、左右乘约定、右手系/左手系。
- 近似是否一致：论文使用一阶近似，代码却引入二阶项（或相反），但未声明。
- 离散化是否一致：连续方程是否被欧拉/Runge-Kutta/半隐式等方法近似，误差阶是否满足需求。

### D. Implementation ↔ Tests

- 测试是否能失败：是否存在“永远通过”的断言（例如只检查非 NaN、只检查长度不为 0）。
- 覆盖关键不变量/基准：
  - 两体能量/角动量守恒（在无摄动、无机动时）。
  - 已知解析解或高精度基准（例如两体 Kepler 解、已验证的对照数据）。
  - 极限情况：e \u2192 0、i \u2192 0、近圆近赤道退化、\u03c1 很小/很大、dt 很小/很大。
- 摄动/机动事件的测试：模型切换点是否覆盖（分段连续性、事件触发条件）。

### E. Cross-Cutting Risks

- 数值稳定性：长期积分能量漂移是否被监控；步长与容差是否与误差预算一致。
- 可复现性：随机数/初始条件/常量版本是否固定；是否依赖外部环境数据而未版本化。
- 观测与滤波：线性化点、Jacobian、残差定义是否一致；是否存在隐含尺度因子。
